# Prelegal

AI-powered legal document drafting platform. Users select a template, chat with an AI assistant to fill in the terms, preview the document in real time, and download a print-ready PDF.

## How It Works

1. **Sign up / Sign in** -- cookie-based authentication with bcrypt password hashing.
2. **Pick a template** -- choose from 12 industry-standard legal document types on the home page.
3. **Chat with AI** -- a split-pane editor opens: AI chat on the left, live document preview on the right. The AI guides you through each field progressively.
4. **Download PDF** -- add electronic signatures via on-screen pads, then generate a pixel-accurate PDF rendered by headless Chromium.
5. **Resume later** -- saved documents appear in "My Documents" on the home page, where you can preview, re-download, or continue editing.

## Tech Stack

| Layer | Technology |
|---|---|
| Frontend | Next.js 14, React 18, TypeScript, Tailwind CSS |
| Backend | Python 3.12, FastAPI, uv |
| AI | LiteLLM via OpenRouter (Cerebras inference, `arcee-ai/trinity-large-preview:free`) |
| Database | SQLite (ephemeral, recreated on container start) |
| PDF | Playwright (headless Chromium) |
| Auth | bcrypt + cookie-based sessions |
| Packaging | Docker multi-stage build (Node + Python) |

## Getting Started

### Prerequisites

- Docker and Docker Compose
- An [OpenRouter](https://openrouter.ai) API key

### Setup

1. Clone the repository:
   ```bash
   git clone https://github.com/Marchenko1997/prelegal.git
   cd prelegal
   ```

2. Create a `.env` file in the project root:
   ```
   OPENROUTER_API_KEY=your_key_here
   ```

3. Start the application:

   **Mac**
   ```bash
   scripts/start-mac.sh
   ```

   **Linux**
   ```bash
   scripts/start-linux.sh
   ```

   **Windows (PowerShell)**
   ```powershell
   scripts/start-windows.ps1
   ```

4. Open [http://localhost:8001](http://localhost:8001) in your browser.

### Stopping

```bash
# Mac
scripts/stop-mac.sh

# Linux
scripts/stop-linux.sh

# Windows (PowerShell)
scripts/stop-windows.ps1
```

## API Endpoints

| Method | Path | Description |
|---|---|---|
| `GET` | `/api/health` | Health check |
| `GET` | `/api/catalog` | List all document types |
| `POST` | `/api/auth/signup` | Create account |
| `POST` | `/api/auth/signin` | Sign in |
| `POST` | `/api/auth/signout` | Sign out |
| `GET` | `/api/auth/me` | Current user info |
| `POST` | `/api/chat` | AI chat (Mutual NDA) |
| `POST` | `/api/doc-chat` | AI chat (generic documents) |
| `POST` | `/api/generate` | Render Mutual NDA to HTML |
| `POST` | `/api/generate-doc` | Render generic document to HTML |
| `POST` | `/api/pdf` | Generate PDF from HTML |
| `POST` | `/api/documents` | Save a document |
| `GET` | `/api/documents` | List user's saved documents |
